## Rules

- NEVER add "Co-Authored-By" or any AI attribution to commits.
- Never build after changes unless explicitly asked.
- When asking a question, STOP and wait for response. Never assume answers.
- Verify technical claims before stating them. If unsure, investigate first.

## Code Style

- Follow existing conventions in the file being edited.
- Prefer explicit over clever — readable code over compact code.
- Use conventional commits format: `feat:`, `fix:`, `refactor:`, `docs:`, `test:`.
- Keep functions small and focused on a single responsibility.

## Language

- Match the language the user writes in.
- Be direct and concise. No filler phrases.
